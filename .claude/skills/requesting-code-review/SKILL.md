# Code Review Request Guidelines

This document establishes a structured approach for requesting code reviews using a superpowers:code-reviewer subagent.

## Key Takeaways

The guidance emphasizes that code review should be "Review early, review often." Reviews are mandatory after completing tasks in subagent-driven development, finishing major features, and before merging to main branches.

The process requires three steps: obtaining git commit hashes, dispatching the code-reviewer subagent with a populated template, and acting on feedback by immediately addressing critical issues and important concerns before proceeding.
